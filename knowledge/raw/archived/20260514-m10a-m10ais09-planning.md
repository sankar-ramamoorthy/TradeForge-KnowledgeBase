# M10AIS09 Planning — Replay Cognitive Artifact Timeline

## Scope

Replay reconstructs reasoning, not merely events.
Surface thesis and plan cognitive artifact content inline within the
replay timeline so operators can reconstruct the operator's thinking
at each lifecycle stage.

## No Backend Changes Required

ADR-0035 confirmed: GET /replay/timeline already returns full event payloads.
The thesis and plan payload data from M10AIS01-06 is already in timeline entries.
This is a frontend-only rendering enhancement.

## Current State

TimelineEntryRow shows: event_type, kind badge, stage badge, sequence, timestamp.
payload field is available but not rendered.

## Implementation

### Type Guards (inline in ReplayWorkspace.tsx)

extractThesisPayload(payload: Record<string, unknown>): ThesisPayloadData | null
extractPlanPayload(payload: Record<string, unknown>): PlanPayloadData | null

ThesisPayloadData: { narrative, catalysts, assumptions, invalidation_conditions, confidence_level, regime_alignment }
PlanPayloadData: { entry_rationale, stop_rationale, target_rationale, sizing_rationale, execution_assumptions, playbook_alignment }

### ThesisPayloadPreview subcomponent

Renders inside TimelineEntryRow for thesis_created and thesis_revised events.
Shows:
- narrative (truncated at 150 chars if needed, with full text on title attr)
- conviction badge (1-5 label)
- catalyst count, invalidation condition count
- regime alignment if set

### PlanPayloadPreview subcomponent

Renders inside TimelineEntryRow for plan_created events.
Shows:
- entry_rationale (truncated at 150 chars)
- playbook badge if set
- execution assumptions count

### CognitiveSnapshotSummary panel

Appears at top of timeline section (before the ol list).
Derives the "latest cognitive state" from all timeline entries:
- finds the last thesis event and extracts its data
- finds the last plan event and extracts its data
- shows compact summary: "Latest Thesis: [narrative excerpt] (3/5)"
  + "Plan: [entry excerpt]" if plan exists
- shows thesis evolution count (N revisions) if thesis_revised events exist
- absent when no cognitive artifacts present (graceful degradation)

## Graceful Degradation

Events with empty payloads (seeded demo scenarios created before M10A) show
no artifact preview — they simply display as before without the artifact section.
No error thrown.

## Testing Strategy

- No new backend tests needed (no backend changes)
- Frontend: typecheck clean, build clean
- Manual verification: create thesis + plan, view replay timeline
  - ThesisPayloadPreview appears for thesis_created entry
  - PlanPayloadPreview appears for plan_created entry
  - CognitiveSnapshotSummary shows at top of timeline

## Architecture Notes

- All logic is presentational — no mutations, no lifecycle changes
- Type guards defensively check payload shape — handles legacy empty payloads
- All artifact content is clearly labeled as "from event payload" to avoid
  confusion with canonical state
- Satisfies ADR-0035: "Consuming surfaces extended to extract and display
  structured artifact content from these payloads where available"
