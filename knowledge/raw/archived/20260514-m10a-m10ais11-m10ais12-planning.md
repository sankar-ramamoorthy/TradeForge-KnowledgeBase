# M10AIS11-12 Planning

## Scope

M10AIS11: ReviewReflectionArtifact domain model + complete-review API endpoint + review query endpoint
M10AIS12: ReviewReflectionModal frontend form + cognition-aware ReviewWorkspace update

## Core Semantic Distinction

ReviewReflection separates:
- decision_quality (was the reasoning process sound? — independent of outcome)
- execution_quality (was the plan followed? — independent of PnL)
- thesis_vs_outcome (narrative: what was expected vs what actually happened)

This is the canonical behavioral learning artifact. "Good process, bad outcome" and
"bad process, good outcome" must be distinguishable.

## ReviewReflectionArtifact Fields

- thesis_vs_outcome: str (required) — what actually happened vs what was expected
- decision_quality: int 1-5 (required) — quality of the reasoning process
- execution_quality: int 1-5 (required) — how well the plan was executed
- discipline_observations: str (required) — notes on process adherence or deviation
- lessons_learned: list[str] (required, min 1) — actionable insights
- behavioral_observations: str (optional) — patterns or tendencies noticed

## Event Model

Creates review.review_completed lifecycle event (Position → Review stage).
This IS a lifecycle transition via LifecycleOrchestrationService.
Structured reflection embedded in event payload: {review: {...}}.

## ADR Checkpoint

No new ADR needed. ADR-0033 (structured cognitive artifact model) covers this.
ADR-0002 (decision lifecycle engine) covers the Position→Review transition.

## Backend Endpoints

POST /lifecycle/decisions/complete-review
- validates all required fields
- creates review.review_completed lifecycle transition
- embeds structured reflection in event payload

GET /lifecycle/decisions/{decision_id}/review
- reads review.review_completed event payload
- returns ReviewReflectionArtifactResponse (with symbol)
- returns 404 for decisions without a completed review

## Frontend Changes

1. ReviewReflectionModal.tsx
   - thesis_vs_outcome (textarea, required)
   - decision_quality (1-5 range slider)
   - execution_quality (1-5 range slider)
   - discipline_observations (textarea, required)
   - lessons_learned (dynamic list, required)
   - behavioral_observations (textarea, optional)
   - submit to POST /lifecycle/decisions/complete-review
   - on success: reload projection (stay in review workspace, now at Review stage)

2. ReviewWorkspace.tsx updates
   - Fetch thesis, plan, and existing review artifacts on load
   - ReviewFoundationPanel: shows original thesis narrative + plan entry rationale
     for cognitive context before writing the reflection
   - Replace handleCompleteReview with modal open
   - ReviewReflectionPanel: shows captured reflection content when reviewComplete

3. ReviewFoundationPanel design
   - Shows BEFORE the action button
   - "Your original thesis: [narrative excerpt]"
   - "Your original plan: [entry rationale excerpt]"
   - This is the replay-aware cognitive context for the reflection

## Key Difference from Previous Modals

Thesis: on success navigate to plan-review
Plan: on success reload projection
Review: on success reload projection (stays in review workspace at Review stage)

## Testing Strategy

Backend:
- POST /lifecycle/decisions/complete-review creates review.review_completed event
- POST /lifecycle/decisions/complete-review rejects empty thesis_vs_outcome
- POST /lifecycle/decisions/complete-review rejects empty lessons_learned
- POST /lifecycle/decisions/complete-review rejects when not in Position stage
- GET /lifecycle/decisions/{id}/review returns reflection content
- GET /lifecycle/decisions/{id}/review returns 404 without a review
- Reflection payload embedded in replay timeline entry

Frontend:
- typecheck clean, build clean
