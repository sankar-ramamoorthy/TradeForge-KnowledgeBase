# M10AIS10 Planning — Cognitive Snapshot Reconstruction

## Scope

Reconstruct operator cognition at any historical timestamp.
Given timestamp T: return the lifecycle stage, most recent thesis, most recent plan,
and all scenario branches as they existed at time T.

## Key Distinction from M10AIS09

M10AIS09: Show individual event payloads inline in the timeline.
M10AIS10: Reconstruct accumulated cognitive state at a point in time.

Example: thesis created at event #2, revised at event #8.
At event #5 timestamp, M10AIS10 reconstructs thesis from event #2
(not the revision from #8, which hadn't happened yet).

## Architecture

Deterministic reconstruction from immutable events.
No live APIs. No external state.
Fully replayable — same result for same input always.

## Backend Endpoint

GET /lifecycle/decisions/{decision_id}/cognitive-snapshot
Optional query param: at (ISO datetime, default = now)

Scans events in chronological order.
For each event referencing this decision AND timestamp <= at:
- track latest lifecycle stage
- track latest thesis payload (thesis_created or thesis_revised)
- track latest plan payload (plan_created)
- collect all scenario branch payloads

Returns CognitiveSnapshotResponse with:
- decision_id
- snapshot_at (the timestamp used)
- event_count_at_snapshot (how many events were visible)
- current_stage (lifecycle stage at T)
- thesis (most recent thesis before T, or null)
- plan (most recent plan before T, or null)
- scenario_branches (all branches before T)
- authority: "derived"

## Derived Response Models (new, compact)

CognitiveSnapshotThesisData — narrative, catalysts, assumptions, invalidation_conditions,
confidence_level, regime_alignment, event_type, event_timestamp

CognitiveSnapshotPlanData — entry_rationale, stop_rationale, target_rationale,
sizing_rationale, execution_assumptions, playbook_alignment, event_timestamp

CognitiveSnapshotBranchData — branch_type, condition, implication, confidence,
notes, event_timestamp

## Frontend

1. CognitiveSnapshotPanel.tsx
   - Compact panel showing lifecycle stage, thesis, plan, branches at a timestamp
   - Header shows "Cognitive Snapshot at [timestamp]" or "Current State" when no selection
   - Thesis: narrative excerpt, conviction badge, catalyst/invalidation counts
   - Plan: entry/stop rationale excerpts, playbook badge
   - Branches: count by type (e.g., "2 primary · 1 invalidation")
   - Authority note: "Derived from event payloads — not canonical truth"

2. ReplayWorkspace.tsx updates
   - selectedEntryTimestamp state (string | null)
   - Clicking a timeline entry sets selectedEntryTimestamp = entry.timestamp
   - When decision_id is available, fetch cognitive snapshot (with or without at param)
   - Show CognitiveSnapshotPanel below CognitiveSnapshotSummary

3. fetchCognitiveSnapshot() API function in runtime.ts

## ADR Checkpoint

No new ADR. ADR-0035 explicitly defines this:
"Point-In-Time Reconstruction: Given a timestamp T, replay reconstructs all lifecycle
events before T, all thesis/plan event payloads before T, the cognitive state as it
would have been visible at time T."

## Testing Strategy

Backend:
- returns current state when no at param
- returns correct thesis at a past timestamp before a revision
- returns null thesis when queried before thesis was created
- returns correct scenario branches at a past timestamp
- returns correct plan when queried after plan was created
- returns correct stage at each lifecycle step
