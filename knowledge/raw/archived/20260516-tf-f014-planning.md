# TF-F014 Planning - Remaining Workspace Zoning Propagation

## Diagnosed Root Cause Summary

`TF-F012` introduced the shared workstation layout primitive and converted the Operating Workspace as the reference implementation. `OpportunityWorkspace` and `ActivePositionWorkspace` still render `MarketContextPanel` inline, so the workstation zoning model is not yet consistently applied across the remaining workspaces that visibly need persistent market context.

## Chosen Approach

Propagate the existing rail model only to the workspaces with clear inline persistent market context:

1. move `MarketContextPanel` out of the primary flow for `OpportunityWorkspace`
2. move `MarketContextPanel` out of the primary flow for `ActivePositionWorkspace`
3. provide the relevant rail content from the app-level workspace shell so each route uses the same three-zone primitive
4. preserve each workspace's distinct primary workflow content

## Why This Approach

This is the smallest follow-on that makes the new workstation layout model materially consistent without forcing every workspace into the same composition.

## Affected Runtime Files

- `TradeForge/frontend/src/App.tsx`
- `TradeForge/frontend/src/workspaces/OpportunityWorkspace.tsx`
- `TradeForge/frontend/src/workspaces/ActivePositionWorkspace.tsx`

## Lifecycle / Event / Replay Implications

- lifecycle implications: none
- event implications: none
- replay implications: none

## Testing Approach

- `npm.cmd run typecheck`
- `npm.cmd run build`
- route-level manual review for:
  - Operating Workspace
  - Opportunity Workspace
  - Active Position Workspace
- confirm advisory/contextual market surfaces remain outside canonical workflow regions

## Regression Risk Areas

- rail duplication across routes
- accidental removal of useful context from a workspace
- responsive stacking regressions
- context rail content appearing on routes where it is not relevant

## Out Of Scope

- new panel types
- review/replay workspace redesign
- layout customization
- lifecycle or event changes

No ADR required.
