# TF-F007 Planning Capture - Credential Setup Documentation

## Issue Scope

- Issue ID: TF-F007
- Milestone: M10C
- Title: Credential setup guide, rotation documentation, and keys-out-of-Git enforcement
- Affected layers: docs, operational
- Linked ADR: ADR-0037

## Architectural Interpretation

TF-F007 closes the operator-facing side of the credential boundary introduced by TF-F004 through TF-F006.

The bounded documentation addition is:

1. A root `HOW-TO-SETUP-KEYS.md` guide.
2. Explicit Git ignore coverage for `.keys.enc` and accidental files named `TRADEFORGE_MASTER_KEY`.
3. A README reference so the setup path is discoverable from the project entrypoint.

## Required Guide Coverage

- Generate `TRADEFORGE_MASTER_KEY` with `scripts/manage_credentials.py`.
- Set the master key in the OS environment only.
- Register Polygon, Alpaca, Alpha Vantage, FMP, and Finqual credentials.
- Select `TRADEFORGE_MARKET_PROVIDER` where relevant.
- Rotate provider credentials and revoke/remove stale credentials operationally.
- State what must never enter Git or logs.

## Event / Replay / Lifecycle Analysis

- Event impact: none.
- Replay impact: documentation should explain why credential status matters operationally without implying event-sourced credential history already exists.
- Lifecycle impact: none.

## Validation Strategy

- Verify guide existence and README linkage.
- Verify `.gitignore` contains both `.keys.enc` and `TRADEFORGE_MASTER_KEY`.
- Keep the guide executable against the actual script surface from TF-F005.

## Out Of Scope

- Changing credential-store behavior.
- Adding provider adapters not yet implemented.
- Introducing a broader CLI framework.
