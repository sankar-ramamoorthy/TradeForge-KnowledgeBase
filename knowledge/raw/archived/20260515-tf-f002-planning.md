# TF-F002 Planning Capture

## Diagnosis Summary

The current lifecycle compresses APPROVAL → EXECUTION.
In real discretionary trading, approval means "authorized if conditions occur" — not
"executing immediately." The SMH plan required daily close above 585, rising volume,
and breadth participation before entry.

No state exists for: "authorized but awaiting trigger conditions."

## Design Decision

ARMED is mandatory between APPROVAL and EXECUTION.
New canonical lifecycle:
  Idea → Thesis → Plan → Approval → Armed → Execution → Position → Review

Rationale: Real entries always have trigger conditions, even "execute at market open."
Making ARMED mandatory enforces declaration of those conditions. ADR 0002 requires
all stages to be explicit — ARMED makes the trigger moment explicit and replayable.

Event type: `decision.plan_armed` (decision domain — execution hasn't started yet)
Payload: trigger_conditions (list[str], min 1 required)

## ADR Required

Classification: architectural — new lifecycle state, new event type, lifecycle engine change.
ADR-0036 to be created: Armed Lifecycle Stage.

## Scope

Backend:
- LifecycleStage.ARMED = "Armed" enum value
- CANONICAL_LIFECYCLE_STAGES updated (8 stages)
- LIFECYCLE_EVENT_STAGE_MAP: "decision.plan_armed" → ARMED
- ALLOWED_LIFECYCLE_TRANSITIONS: APPROVAL→ARMED, ARMED→EXECUTION
- LIFECYCLE_STAGE_EVENT_TYPE_MAP: ARMED → "decision.plan_armed"
- POST /lifecycle/decisions/arm-plan endpoint
- GET /lifecycle/decisions/{id}/arm endpoint (read trigger conditions)
- Pydantic models: ArmPlanPayload, ArmPlanResponse, ArmPlanArtifactResponse

Frontend:
- LifecycleProgress.tsx: add "Armed" stage between Approval and Execution
- workspaceRouting.ts: Armed → active-position
- PlanReviewWorkspace: replace "Record Execution" at Approval with "Arm Plan" modal
- ArmPlanModal.tsx: new modal for trigger condition declaration
- ActivePositionWorkspace: Armed state UI with trigger conditions + "Confirm Trigger Met"

Tests:
- Update test_lifecycle_transitions.py (canonical map, valid/invalid transitions)
- Update test_lifecycle_state.py (CANONICAL_LIFECYCLE_STAGES, stage count)
- Update test_mvp_lifecycle_flow.py (add Armed between Approval and Execution)
- New integration tests for arm-plan endpoint

Out of scope:
- Armed → Approval reversal (cancel arming — possible future enhancement)
- Multiple trigger conditions system / automated trigger monitoring
- Optional ARMED path (direct APPROVAL → EXECUTION) — might revisit if UX friction too high
