# TF-F006 Planning Capture - Provider Credential Wiring

## Issue Scope

- Issue ID: TF-F006
- Milestone: M10C
- Title: Wire all provider adapters through CredentialStore at composition root
- Affected layers: app, infrastructure, security
- Linked ADR: ADR-0037
- Impacted invariants: Architectural Simplicity, Layer Separation

## Architectural Interpretation

TF-F006 connects the credential boundary to runtime composition without changing provider adapter ownership.

The bounded runtime change is:

1. `create_app()` accepts `credential_store: CredentialStore | None`.
2. Market-provider construction is centralized inside the composition root.
3. `yfinance` remains keyless and default.
4. Credentialed providers receive decrypted constructor fields only inside `src/app/api/application.py`.
5. Existing provider adapters remain ignorant of `src/security/`.

## Composition Design

- `TRADEFORGE_MARKET_PROVIDER` selects the default provider id; default is `yfinance`.
- If `.keys.enc` exists and no store is injected, the composition root may initialize `CredentialStore()`.
- A single helper resolves a provider id to the proper adapter constructor.
- Credentialed providers decrypt payloads using `KeyManager.from_environment()` and require the credential record to exist.
- Future adapters should be added to the same provider-construction helper so they inherit the credential path by default.

## Data Flow

```text
create_app()
    -> provider id resolution
    -> CredentialStore.get(provider_id)
    -> KeyManager.decrypt_payload()
    -> provider constructor args
    -> MarketSnapshotService(provider)
```

## Event / Replay / Lifecycle Analysis

- Event impact: none.
- Replay impact: none beyond preserving explicit provider capability boundaries already modeled in credential metadata.
- Lifecycle impact: none.

## Validation Strategy

- Keep provider adapters free of `src.security` imports.
- Add integration coverage using `create_app(credential_store=...)` through `/workspaces/market-context`.
- Assert credentialed providers are constructed with decrypted fields while route behavior remains unchanged.

## Out Of Scope

- Adding new provider adapters for Alpha Vantage, FMP, or Finqual.
- Operator-facing credential documentation.
- Replacing adapter constructor injection in adapter tests.
