# TF-F027 Planning - Provider Capability Rail Clarity

## Approach

- Extend `ProviderConfigurationPanel` so it renders both `price` and `fundamentals` capability sections.
- Keep fundamentals editable through the existing selector.
- Render the selected price provider and fallback order explicitly so the adjacent Market Context panel is visibly tied to price capability.
- Leave backend services and capability resolution semantics unchanged.

## Rationale

The operator-facing gap is a misleading presentation of already-correct architecture. The smallest correct fix is to make the separate capability paths visible at the point where the operator encounters them.

## Replay / Lifecycle Impact

None.

## Regression Focus

- provider configuration rendering still works when fundamentals resolution is present
- frontend type safety and production build remain clean
