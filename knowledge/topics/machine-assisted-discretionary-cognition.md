# Machine-Assisted Discretionary Cognition

Machine-assisted discretionary cognition is the emerging roadmap frame for post-M11 TradeForge.

It means TradeForge uses machine assistance to improve discretionary human reasoning without transferring decision authority to the machine.

Machine assistance may support:

- broad market attention
- candidate discovery
- evidence accumulation
- contextual interpretation
- anomaly detection
- historical comparison
- behavioral pattern surfacing
- replayable cognitive reconstruction
- attention allocation

The operator retains authority over:

- judgment
- discretionary interpretation
- risk acceptance
- lifecycle commitment
- plan approval
- execution decisions
- accountability

## Boundary

Machine-originated artifacts belong to advisory space until an operator explicitly promotes them through the decision lifecycle.

Advisory artifacts must not automatically become TradeIdeas, theses, plans, approvals, executions, or positions.

## LLM Adapter Sequencing

Post-M12 advisory infrastructure became operationally alive in M13 through a
concrete OpenAI-compatible LLM adapter routed through LiteLLM.

The original sequencing rule was:

```text
concrete advisory LLM adapter -> validated prompts -> M13 interpretation semantics
```

M13 completed that sequence at the runtime level: LiteLLM credential shape,
OpenAI-compatible provider, on-demand advisory services, interpretation
analytics, and replay/UX API surfaces are implemented.

The preferred adapter strategy is a single OpenAI-compatible provider routed
through LiteLLM. Provider selection remains configuration and credential
management, not domain semantics.

## Roadmap Role

This topic frames the active v2 roadmap from M12 onward:

- M12: advisory observations and cognitive evidence
- M13: contextual interpretation and thesis influence (runtime complete)
- M14: behavioral intelligence and cognitive auditability (active; TF-C001 complete)
- M15: replayable cognitive reconstruction
- M16: attention allocation and opportunity funnel
- M17: simulation and regime experimentation
- M18: adaptive advisory research infrastructure
- M19: long-horizon cognitive performance analysis

## M14 Behavioral Signal Boundary

M14 begins with deterministic BehavioralSignals rather than opaque behavioral
scores or AI-authored judgments.

The first stable boundary is:

```text
Event Ledger history -> deterministic detector -> BehavioralSignal
```

BehavioralSignals are derived, replayable, source-linked review context. They do
not create lifecycle authority, execution authority, or canonical behavioral
events.
