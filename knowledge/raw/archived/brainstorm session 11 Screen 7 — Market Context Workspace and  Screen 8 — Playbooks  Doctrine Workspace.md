Yes — for MVP v1, I think there are really **two remaining major screens** to complete the operational loop.

You already have the core lifecycle:

| Screen                       | Purpose                          |
| ---------------------------- | -------------------------------- |
| 1. Operating Workspace       | Operational command center       |
| 2. Opportunity Workspace     | Develop structured hypotheses    |
| 3. Plan Review Workspace     | Authorize intentional risk       |
| 4. Active Position Workspace | Manage live decision state       |
| 5. Replay Workspace          | Reconstruct cognition            |
| 6. Review Workspace          | Convert experience into learning |

Those six screens already form a coherent product.

But two additional screens complete the ecosystem and make the system feel operationally whole for a swing trader.

---

# Remaining Screen 7 — Market Context Workspace

This is the:

```text id="mc1"
environment-awareness screen
```

Purpose:

```text id="mc2"
“What environment am I trading inside?”
```

This becomes the contextual intelligence layer.

---

# Why It Matters

Swing trading performance is massively dependent on:

* regime
* breadth
* leadership
* volatility
* macro context
* liquidity
* earnings environment

Most traders incorrectly evaluate trades in isolation.

TradeForge should continuously remind the trader:

```text id="mc3"
context matters
```

---

# Remaining Screen 8 — Playbooks / Doctrine Workspace

This is probably the most underestimated screen.

Purpose:

```text id="pb1"
“What rules govern my operation?”
```

This screen defines:

* how the trader operates
* what setups exist
* what risk means
* what invalidates trades
* what constitutes discipline

---

# Why It Matters

Without this screen:

TradeForge risks becoming:

```text id="pb2"
ad hoc discretionary chaos
```

With this screen:

TradeForge becomes:

```text id="pb3"
explicit operational doctrine
```

---

# This Screen Is NOT

## NOT Generic Settings

Not:

* profile page
* preferences page
* theme picker

---

# This Screen IS

```text id="pb4"
cognitive operating doctrine
```

---

# Likely Sections

## Playbooks

Example:

```text id="pb5"
• Continuation Breakout
• Pullback Trend Entry
• Failed Breakdown Reversal
• Earnings Momentum
```

Each playbook defines:

* setup conditions
* regime fit
* invalidation
* execution behavior
* sizing guidance
