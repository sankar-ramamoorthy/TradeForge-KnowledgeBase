# TF-F078 Planning - Evidence API Vite Proxy Gap

## Diagnosed Root Cause

The backend EV routes are present. The dev UI receives a non-JSON response
because Vite lacks a proxy rule for `/evidence`, so the request is answered by
the frontend HTML fallback. The proxy also lacks `/market`, an existing backend
API prefix.

## Chosen Approach

1. Update `frontend/vite.config.ts`:
   - add `/evidence`
   - add `/market`
2. Add a focused regression test in `tests/test_frontend_root_scripts.py`.
   The test should inspect static `fetch("/prefix...")` calls in
   `frontend/src/api/runtime.ts` and assert the matching prefix appears in
   `frontend/vite.config.ts`.
3. Update `DOCS/ISSUE_REGISTER.md` and the M-EZ roadmap note.

## Scope Boundary

This is dev tooling configuration. No backend routes, evidence semantics,
event taxonomy, lifecycle behavior, or replay behavior should change.

## Verification Plan

- `uv run pytest tests/test_frontend_root_scripts.py`
- `npm run typecheck`
- `npm run build`

The original feedback closes when `/evidence/refresh/run` is covered by Vite
proxy configuration and the regression test would fail if the prefix were
removed.
