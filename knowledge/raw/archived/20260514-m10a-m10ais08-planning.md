# M10AIS08 Planning — Plan Validation Preview Layer

## Scope

Surface cognition readiness and lifecycle rule previews before the operator
authorizes a plan. Appears in PlanReviewWorkspace at Plan stage.

## Design Decision: Backend vs Frontend

Backend endpoint chosen over frontend-only logic because:
- Validation logic is deterministic and replayable
- Separates readiness reasoning from presentation
- Can be tested independently of the UI
- Follows existing pattern of derived advisory endpoints

## No ADR Required

This is a derived advisory surface. No new architectural boundaries.
ADR-0033 (cognitive artifact model) and ADR-0004 (workspace projection model)
already cover derived advisory surfaces.

## Readiness Check Categories

### Hard Gates (blocking — advisory label "Required")
- has_structured_thesis: decision.thesis_created event has structured thesis payload
- has_structured_plan: decision.plan_created event has structured plan payload

### Advisory Checks (informational — label "Advisory")
- conviction_level: thesis conviction_level value (warn if < 3)
- invalidation_conditions: count of defined conditions
- execution_assumptions: count of defined assumptions
- playbook_alignment: whether a playbook is named

### Lifecycle Readiness (informational)
- current_stage: what stage is the decision at?
- next_allowed_transition: what transition comes next?

## Endpoint

GET /lifecycle/decisions/{decision_id}/plan-readiness

Returns:
- decision_id
- current_stage (str or null)
- next_allowed_transition (str or null)
- has_structured_thesis (bool)
- has_structured_plan (bool)  
- can_proceed_to_approval (bool — true when all hard gates pass AND stage is Plan)
- checks: list of ReadinessCheckResponse
- authority: "derived"

ReadinessCheckResponse:
- check_id: str
- label: str
- passed: bool
- advisory: bool
- message: str

## Frontend Component

PlanReadinessPanel.tsx
- Appears in PlanReviewWorkspace when lifecycle stage = Plan
- Fetched via GET /lifecycle/decisions/{id}/plan-readiness
- Shows overall readiness status (ready / needs attention)
- Lists all checks with visual indicators
- Compact panel — does not replace action buttons

PlanReviewWorkspace.tsx:
- Add readinessData state
- Fetch plan-readiness when plan artifact is fetched (at Plan stage)
- Render PlanReadinessPanel above "Authorize Plan" button

## Testing Strategy

Backend:
- returns can_proceed_to_approval=true when thesis+plan both structured
- returns can_proceed_to_approval=false when only legacy thesis (empty payload)
- returns can_proceed_to_approval=false at wrong lifecycle stage
- conviction_level advisory check warns when confidence_level < 3
- returns current_stage and next_allowed_transition correctly
- returns 404 when decision not found in event store

Frontend:
- typecheck clean, build clean
