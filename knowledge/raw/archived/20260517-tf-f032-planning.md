# TF-F032 Planning

## Issue

- Issue ID: `TF-F032`
- Milestone: `M10E`
- Title: Add Explicit Advisory Context Acquisition Workflow
- Affected layers: services, app, frontend
- Impacted ADRs: `ADR-0010`, `ADR-0038`, `ADR-0039`, `ADR-0040`
- Impacted invariants: UX Is Architectural, Workflow-Centric Architecture

## Design Intent

Implement the first bounded runtime form of the `Context Workbench` as an
explicit advisory acquisition workflow rather than adding more hidden loading to
the Opportunity Workspace.

## Planned Runtime Shape

- Add a first-class `context-workbench` workspace route.
- Define an advisory-only workspace state contract with no lifecycle actions.
- Add a dedicated frontend workspace for one instrument at a time.
- Let the operator request available context families independently:
  - price / technical context
  - fundamentals
- Surface acquisition state per family:
  - not requested
  - loading
  - loaded
  - unavailable
- Keep provider configuration separate from acquisition.

## Explicit Boundaries

- No lifecycle transition ownership.
- No automatic acquisition of every family.
- No provider attempt detail yet; that remains `TF-F033`.
- No interpretation-first presentation yet; that remains `TF-F042`.
- No broad migration of Opportunity Workspace behavior yet.

## Event Impact

No new canonical events are introduced. The workbench requests advisory overlays
from existing runtime services and does not append lifecycle or decision events.

## Lifecycle Impact

No lifecycle impact. The workbench is pre-decision research context and remains
outside the strict decision lifecycle authority path.
