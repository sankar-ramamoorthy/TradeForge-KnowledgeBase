# Screen 4 — Active Position Workspace

This is where the system transitions from:

```text id="ap1"
planned risk
```

to:

```text id="ap2"
lived reality
```

This is an extremely important distinction.

Because once a trade becomes live:

* emotions begin
* uncertainty increases
* narratives drift
* discipline degrades
* market conditions evolve
* the original thesis is tested

Most trading platforms reduce this phase to:

```text id="ap3"
PnL monitoring
```

TradeForge should instead make this:

```text id="ap4"
decision-state management
```

That is the entire philosophy of Screen 4.

---

# Core Purpose Of The Screen

The screen answers:

```text id="ap5"
“What is happening to this live decision?”
```

NOT merely:

```text id="ap6"
“How much money am I making?”
```

---

# This Is NOT

## NOT A Brokerage Position Monitor

Avoid:

* giant PnL numbers
* flashing ticks
* dopamine coloring
* hyperactive charts
* execution obsession

---

# This IS

```text id="ap7"
a live cognition management workspace
```

This is where:

* thesis integrity
* discipline
* context drift
* execution behavior
* emotional behavior
* risk evolution

are continuously monitored.

---

# Psychological Purpose

The screen should help prevent:

* emotional attachment
* revenge behavior
* thesis drift blindness
* stop movement
* over-management
* panic exits
* greed extensions

The UI should gently ask:

```text id="ap8"
“Does this trade still deserve to exist?”
```

---

Probably:
