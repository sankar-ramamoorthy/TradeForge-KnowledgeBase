# TF-F058 Planning - Root npm run dev ENOENT

## Diagnosed Root Cause

The root repository lacks `package.json`, so npm cannot resolve `npm run dev` when the command is run from the root. The actual frontend package exists under `frontend/`.

## Chosen Approach

Create a root `package.json` containing only operational scripts that delegate to `frontend/` through `npm --prefix frontend run ...`.

## Files To Change

- `package.json`
- `README.md`
- `tests/test_frontend_root_scripts.py`
- `DOCS/ISSUE_REGISTER.md`

## Scope Boundary

The root package must not own frontend dependencies. Dependencies stay in `frontend/package.json` and `frontend/package-lock.json`.

## Event Impact

None.

## Replay Impact

None.

## Lifecycle Impact

None.

## Testing Approach

- Add pytest coverage for root package script shape.
- Run `npm.cmd run typecheck` from the root.
- Run `npm.cmd run build` from the root.
- Run `npm.cmd run dev -- --help` from the root to prove the command resolves without starting a long-lived dev server.
