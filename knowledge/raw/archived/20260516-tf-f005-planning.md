# TF-F005 Planning Capture - Encrypted Local Credential Store

## Issue Scope

- Issue ID: TF-F005
- Milestone: M10C
- Title: Implement KeyManager and encrypted local credential store
- Affected layers: security, infrastructure
- Linked ADR: ADR-0037
- Impacted invariants: Architectural Simplicity, Historical Integrity

## Architectural Interpretation

TF-F005 operationalizes the credential boundary without yet wiring it into provider adapters.

The bounded runtime addition is:

1. A `KeyManager` that loads `TRADEFORGE_MASTER_KEY` from the OS environment and performs Fernet encryption/decryption.
2. A `CredentialStore` that serializes credential metadata plus encrypted payloads to `.keys.enc` as a local encrypted registry.
3. A small operator script that can generate a Fernet master key and register the initial supported provider credentials.
4. Git hygiene for `.keys.enc`.

## Data Flow

```text
operator command
    -> plain credential fields
    -> KeyManager.encrypt_payload()
    -> Credential(encrypted_payload=...)
    -> CredentialStore.save()
    -> .keys.enc
```

Read path:

```text
.keys.enc
    -> CredentialStore.list()/get()
    -> Credential metadata + encrypted payload
    -> KeyManager.decrypt_payload()
    -> provider field mapping
```

## Boundary Decisions

- `KeyManager` and `CredentialStore` stay inside `src/security/` per ADR-0037.
- The store owns file IO, but not provider composition.
- Provider adapters remain unchanged and continue to accept constructor injection only.
- The file stores encrypted payloads, never plaintext provider values.
- Error messages must avoid echoing keys or decrypted payloads.

## Event / Replay / Lifecycle Analysis

- Event impact: none. Credential storage is not event-sourced in TF-F005.
- Replay impact: credential metadata fields remain available for future status-history work.
- Lifecycle impact: none.

## Implementation Plan

- Add `cryptography` dependency.
- Add `.keys.enc` to `.gitignore`.
- Add `src/security/key_manager.py`:
  - explicit `MasterKeyNotConfiguredError`
  - `from_environment()` loader for `TRADEFORGE_MASTER_KEY`
  - `generate_master_key()` helper
  - JSON-based payload encryption/decryption using Fernet
- Add `src/security/credential_store.py`:
  - deterministic JSON file shape
  - write/read/list/get methods
  - status filtering
  - bytes encoded safely for JSON persistence
- Extend `src/security/__init__.py` exports.
- Add `scripts/manage_credentials.py` using `argparse`:
  - `generate-master-key`
  - `register` for polygon, alpaca, alpha_vantage, fmp, finqual
- Add focused tests for KeyManager and CredentialStore.

## Validation Against Invariants

- Architectural simplicity: one local encrypted registry, no external secrets service.
- Historical integrity: credential metadata remains explicit and durable.
- Event sourcing integrity: no hidden workflow state introduced; this remains an operational capability registry, not canonical decision truth.
- Layer separation: no provider adapter imports from `src/security/`.

## Out Of Scope

- Composition-root wiring (`TF-F006`)
- Operator guide and rotation docs (`TF-F007`)
- Event-sourced credential lifecycle history
