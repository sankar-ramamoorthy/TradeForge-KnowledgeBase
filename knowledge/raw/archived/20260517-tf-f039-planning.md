# TF-F039 Planning - Missing Information Doctrine

## Goal

Define a reusable UX rule for missing-information states so TradeForge surfaces
do not stop at technically truthful but cognitively incomplete messages.

## Architectural Interpretation

- Missing information is part of uncertainty handling, not a cosmetic empty state.
- Operator-facing surfaces must explain meaning, continuity, and recovery.
- The doctrine should be broad enough to govern future context, provider, and
  workspace surfaces without implementing each one now.

## Planned Changes

1. Add canonical missing-information doctrine in the KB design layer.
2. Extend `UX_DOCTRINE.md` with the recovery-oriented rule.
3. Extend `frontend/DESIGN.md` with the runtime translation requirement.
