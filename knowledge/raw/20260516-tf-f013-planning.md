# TF-F013 Planning - Three-Layer Design Architecture

## Diagnosed Root Cause Summary

The documentation model has not caught up with the emergence of a distinct operational workspace architecture layer between high-level design doctrine and frontend implementation translation.

## Chosen Approach

Clarify the design hierarchy first, without prematurely canonicalizing draft documents.

### Required Documentation Change

Update the frontend design role note so it explicitly describes:

```text
cognitive doctrine
operational workspace architecture
frontend implementation translation
```

### Required Boundary Clarification

State explicitly that `TradeForge/frontend/DESIGN.md`:

- owns tokens and implementation primitives
- does not own workspace meaning
- does not own operational composition doctrine
- translates stable doctrine into visible UI behavior

### Promotion Discipline

Keep the newer workspace-layout documents in draft state unless a separate promotion decision is made.

The issue should clarify the relationship now, not silently promote all related artifacts.

## Why This Approach

The current gap is interpretive rather than structural.

A small doctrine clarification should reduce future frontend drift while preserving the repository's progressive discovery model.
