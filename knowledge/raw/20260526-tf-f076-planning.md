# TF-F076 LiteLLM Readiness Healthcheck Planning

## Issue

TF-F076: Replace LiteLLM route-probing healthcheck with non-invasive readiness check.

## Scope

- Add TF-F076 as an M13B corrective feedback issue.
- Replace the LiteLLM Docker healthcheck endpoint from `/health` to `/health/readiness`.
- Preserve LiteLLM internal-only exposure.
- Preserve `LITELLM_MASTER_KEY` for the LiteLLM service.
- Do not add downstream provider API keys to Compose.
- Do not change wildcard LiteLLM routes in this issue.
- Add tests that prevent the healthcheck from regressing to `/health`.

## Event And Lifecycle Impact

None. This is an operational readiness fix. It does not create events, mutate canonical state, affect lifecycle authority, or change advisory artifact semantics.

## Verification Plan

- Run `uv run pytest tests/test_postgres_compose.py`.
- Run `docker compose --profile advisory config`.
- If a live advisory profile is available, observe LiteLLM idling across multiple healthcheck intervals and confirm no provider fallback attempts are emitted solely from Docker health checks.
