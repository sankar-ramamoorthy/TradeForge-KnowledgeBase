# TF-F026 Planning - Compose Master Key Forwarding Fix

## Approach

- Add `TRADEFORGE_MASTER_KEY` to the `tradeforge` service environment in `docker-compose.yml`.
- Use required-variable interpolation so Compose reports a missing host key before the runtime process starts.
- Update `HOW-TO-SETUP-KEYS.md` to explain that the shell launching Compose must expose the master key.
- Keep the fix within infrastructure/docs; do not alter `KeyManager`, `CredentialStore`, or provider adapters.

## Rationale

The credential boundary already behaves correctly inside a process. The defect is at the host-to-container handoff, so the minimum correct fix is to make that boundary explicit rather than weaken the credential model.

## Replay / Lifecycle Impact

None.

## Regression Focus

- Compose rendering succeeds when the host key is present.
- Compose rendering fails early when the host key is missing.
- Existing credential behavior remains unchanged.
