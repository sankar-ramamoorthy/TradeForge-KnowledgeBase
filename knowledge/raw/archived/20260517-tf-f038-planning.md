# TF-F038 Planning - Context Workbench Concept

## Goal

Evaluate and define whether a dedicated workspace should own advisory context
acquisition, inspection, and interpretation before later opportunity or thesis
work.

## Architectural Interpretation

- Opportunity evaluation and research acquisition are different operator tasks.
- The existing canonical Research Workspace concept in `WORKSPACES.md` supports
  this distinction semantically.
- A dedicated Context Workbench can become the concrete runtime expression of
  that research-oriented workspace role.

## Planned Changes

1. Create ADR accepting the Context Workbench as a new operational workspace
   concept.
2. Add canonical design document defining its operational question, boundaries,
   inputs, outputs, and exclusions.
3. Keep implementation out of scope for this issue.

## ADR Checkpoint

ADR required because this introduces a durable workspace concept and alters the
workspace architecture set.
