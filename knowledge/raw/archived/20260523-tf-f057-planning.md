# TF-F057 Planning - LiteLLM Compose Runtime Service

## Diagnosed Root Cause

The advisory provider exists, but the local Compose runtime does not host the LiteLLM proxy that the provider talks to. This makes AI advisory setup dependent on an out-of-band process.

## Chosen Approach

Add a `litellm` service to `docker-compose.yml` under an optional Compose profile named `advisory`. Use the official LiteLLM Docker image and mount a repository-owned `litellm_config.yaml` that contains model aliases but no secrets. Provider API keys are read from operator environment variables.

The service exposes `4000:4000` so host-run TradeForge can use `http://localhost:4000`. When TradeForge itself runs in Compose, operators should register the `litellm` credential with `http://litellm:4000`.

## Files To Change

- `docker-compose.yml`
- `litellm_config.yaml`
- `README.md`
- `HOW-TO-SETUP-KEYS.md`
- `tests/test_postgres_compose.py` or a focused compose test
- `DOCS/ISSUE_REGISTER.md`

## Scope Boundary

This change should not alter `OpenAICompatibleAdvisoryProvider`, advisory request/response contracts, lifecycle services, event stores, replay logic, or frontend advisory surfaces.

## Event Impact

None. No event schemas, event types, or append behavior change.

## Replay Impact

None. LiteLLM remains an external advisory service. Replay continues to reconstruct from events and replay-safe advisory capture facts, not live model output.

## Lifecycle Impact

None. LiteLLM cannot approve plans, execute trades, or transition lifecycle state.

## Testing Approach

- Focused pytest checks that Compose defines the optional LiteLLM service, port mapping, config mount, and profile.
- Focused pytest checks that `litellm_config.yaml` references environment variables rather than literal provider keys.
- `docker compose config` to validate Compose syntax.
- Documentation grep/check by inspection.

## Open Questions

- Whether to pin a specific LiteLLM release tag later for supply-chain stability. For this issue, follow current official documentation and keep version pinning as a possible follow-on.
